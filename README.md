
### 3、七大原则：

- ##### 开闭原则：

  ```tex
  一个类只负责一个功能中的职责，就一个类而言,应该只有一个引起他变化的原因。
  原则就是越纯粹，越容易复用。高内聚、低耦合实现的指导方针。
  ```

- ##### 里氏替换原则：

  ```
  引用父类的地方，必须能透明的使用子类对象。
  ```

- ##### 依赖倒置原则：

  ```
  抽象不该依赖细节，细节该依赖抽象。就是面向接口编程，不面向实现编程。
  eg: Spring的依赖注入就是符合这个思想:
  我们写代码时引用到其他类，不引用实现类，直接引用抽象接口或者类，
  依赖注入根据配置文件会帮我们去找具体的实现类。
  我们之后功能变动无需改代码，只需扩展抽象接口的实现，修改配置文件。
  Spring依赖注入里用了开闭原则、里氏代换、依赖倒转三个原则。
  
  ```

- ##### 单一职责原则：

  ```
  一个类只负责一个功能中的职责，就一个类而言,应该只有一个引起他变化的原因。
  原则就是越纯粹，越容易复用。高内聚、低耦合实现的指导方针。
  但是挺难用好的，粒度的把握来自经验。
  ```

- ##### 接口隔离原则：

  ```
  就是接口不能太大，里面方法过多，我用这个接口其实只用一部分方法，导致所有方法我实现类都要实现，
  很不方便。应该拆成多个接口。
  ```

- ##### 迪米特法则：

  ```
  尽可能减少一个类和其他类的相互作用。
  解耦，尽量引入一个中间类来帮助他们交互。
  ```

  

- ##### 合成服用原则：

  ```
  尽量使用对象组合，而不是继承来达到复用的目的。
  因为扩展时，父类发生改变，子类也要发生改变。不灵活，要慎用继承。
  ```
